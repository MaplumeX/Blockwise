# Blockwise - 柳比歇夫时间管理 App 产品需求文档

## 1. 产品概述

### 1.1 产品定位
Blockwise 是一款基于柳比歇夫时间管理理念的安卓应用，通过精确的时间记录和科学的统计分析，帮助用户建立高效的时间管理习惯。

### 1.2 目标用户
- 希望提升时间利用效率的知识工作者
- 需要精确统计工作时间的自由职业者
- 追求自我提升的学生群体
- 实践时间管理方法论的效率爱好者

### 1.3 核心价值
- **精确记录**：细粒度的时间使用记录，了解真实的时间消耗
- **科学统计**：多维度数据分析，发现时间使用规律
- **持续改进**：通过目标管理实现时间使用优化

## 2. 柳比歇夫时间管理核心理念

柳比歇夫时间管理法的核心特点：
1. **精确记录**：记录每项活动的实际消耗时间（精确到分钟）
2. **客观统计**：基于真实数据进行统计分析，避免主观估计
3. **长期追踪**：通过长期数据积累发现时间使用模式
4. **持续改进**：通过目标管理和数据反馈实现时间使用优化

## 3. 用户场景

### 3.1 场景一：学生的一天
**用户**：大学生小李，准备考研

**使用流程**：
1. **早晨**：查看今日目标，确认学习任务
2. **学习时**：使用计时器开始记录"学习"，带标签"英语"
3. **中午**：切换到时间块视图，直观看到上午的时间分布，发现实际学习2.5小时
4. **下午**：继续记录其他学习活动
5. **晚上**：查看统计页面，本周"学习"类已完成18小时，距离目标还差2小时
6. **反思**：在周视图中对比本周和上周的学习分布，发现学习时长增加15%

### 3.2 场景二：自由职业者的项目管理
**用户**：设计师小王，同时进行多个项目

**使用流程**：
1. 为每个项目创建专属标签（项目A、项目B、项目C）
2. 每次工作时精确记录时间
3. 使用周视图查看本周各项目的时间分布，一目了然地看到时间分配比例
4. 月末通过标签统计查看各项目的实际工时
5. 为下月设定目标："项目A不少于60小时，项目C不多于20小时"
6. 通过自定义对比，分析本月 vs 上月的时间使用变化

### 3.3 场景三：职场人士的效率提升
**用户**：程序员小张，想提升深度工作时间

**使用流程**：
1. 创建"深度工作"和"浅度工作"标签
2. 每次工作时使用计时器记录
3. 设定目标：每天深度工作4小时
4. 在日视图中查看时段分布，发现自己在上午9-11点效率最高
5. 根据统计结果，优化工作安排，将重要任务放在高效时段
6. 通过按天对比和时间块视图，追踪每日深度工作时长和分布变化

## 4. 非功能需求

### 4.1 性能要求
- 应用启动时间 < 2秒
- 时间记录保存响应 < 0.5秒
- 统计图表加载 < 3秒
- 支持至少10,000条时间记录的流畅运行

### 4.2 数据安全
- 本地数据加密存储
- 支持数据导出（JSON/CSV格式）
- 支持数据导入和迁移
- 自动备份功能
- 云端备份（可选，需用户授权）

### 4.3 兼容性
- 支持 Android 8.0 (API 26) 及以上版本
- 适配不同屏幕尺寸（手机、平板）
- 支持深色模式
- 多语言支持（简体中文、English）

### 4.4 易用性
- 首次使用引导教程
- 简洁直观的UI设计
- 操作反馈明确（Toast、动画）
- 离线可用（所有核心功能）

## 5. 技术架构建议

### 5.1 推荐技术栈
- **UI层**：Jetpack Compose（现代化声明式UI）
- **架构**：MVVM + Clean Architecture
- **数据库**：Room（本地数据持久化）
- **依赖注入**：Hilt
- **异步处理**：Kotlin Coroutines + Flow
- **图表库**：MPAndroidChart 或 Compose Charts

### 5.2 模块划分
```
app/
├── data/          # 数据层（数据库、数据源）
├── domain/        # 业务逻辑层（用例、实体）
├── presentation/  # 展示层（UI、ViewModel）
│   ├── record/    # 时间记录模块
│   ├── stats/     # 统计模块
│   └── goal/      # 目标模块
└── common/        # 通用工具和组件
```

### 5.3 数据模型设计要点
- TimeEntry（时间记录）为核心实体
- 活动类型支持层级结构（父类型-子类型）
- 统计数据预计算缓存，提升性能

## 6. 成功指标

### 6.1 用户指标
- 日活跃用户留存率 > 40%
- 平均每日记录条数 > 5
- 用户平均使用时长 > 3个月

### 6.2 功能指标
- 时间记录准确率（用户反馈）
- 目标完成率
- 功能使用率（各模块的活跃度）

### 6.3 质量指标
- 崩溃率 < 0.5%
- 应用评分 > 4.3/5.0
- 性能得分 > 85/100

## 7. v1.0 版本功能需求

### 7.1 时间记录（Time Entry）

#### 7.1.1 功能定位

时间记录是对实际发生的活动进行精确记录，是柳比歇夫时间管理法的核心。

#### 7.1.2 记录方式

**手动记录**：

- 选择活动类型、开始时间、结束时间
- 添加标签和备注
- 支持批量导入历史记录

**计时器记录**：

- 一键开始计时
- 实时显示已用时长
- 支持暂停/继续
- 停止时自动创建记录

#### 7.1.3 时间线视图

**视图特性**：

- 按时间顺序展示所有时间记录
- 时间轴形式可视化显示
- 区分不同活动类型的颜色
- 显示每个活动的时长

**编辑功能**：

- 点击记录可编辑所有属性
- 支持删除记录
- 支持拆分记录（一条记录拆为多条）
- 支持合并记录（相同标签的连续记录）

#### 7.1.4 时间块视图

**需求描述**：以时间块（Block）的形式可视化展示已记录的时间，类似日历视图，帮助用户直观了解时间使用分布。

**视图特性**：

- **日视图**：
  - 24小时时间轴，横向或纵向布局
  - 每条时间记录显示为色块，长度对应时长
  - 色块颜色根据活动类型自动分配
  - 显示活动类型和时长
  - 空白区域表示未记录时间
- **周视图**：
  - 7天横向排列，每天一列
  - 每条记录显示为色块
  - 可快速对比工作日和周末的时间使用
  - 支持左右滑动查看不同周

**交互功能**：

- 点击时间块可查看/编辑该条记录
- 长按时间块快速删除
- 拖拽时间块可调整开始时间和时长
- 在空白区域点击可快速创建新记录
- 缩放手势调整视图密度

**导航**：

- 快速跳转到今天/本周
- 左右滑动或使用日期选择器切换日期
- 显示当前视图的统计摘要（总时长、记录数）

**视觉设计**：

- 时间块圆角设计，提升美观度
- 不同标签使用可区分的配色方案
- 时间块内只显示活动类型，节省空间

### 7.2 统计分析（Statistics）

#### 7.2.1 功能定位

基于精确的时间记录数据，提供多维度、多时间跨度的统计分析，帮助用户发现时间使用规律，为时间优化提供数据支撑。

#### 7.2.2 时间维度

**日统计（Daily）**：

- 查看单日的时间分配情况
- 显示当日总记录时长、记录条数
- 支持选择任意日期查看历史数据
- 显示当日目标完成情况

**周统计（Weekly）**：

- 按自然周（周一至周日）统计
- 显示每周总时长和各标签时长
- 支持工作日和周末对比分析
- 显示本周 vs 上周的变化趋势

**月统计（Monthly）**：

- 按自然月统计
- 显示每月总时长和各标签时长分布
- 逐日趋势展示（某标签每天的时长变化）
- 月度目标完成情况汇总

**年统计（Yearly）**：

- 按自然年统计
- 显示全年总时长和各标签年度投入
- 按月展示全年趋势
- 年度时间使用总结报告

**自定义时间范围**：

- 用户自选起始和结束日期
- 支持任意时间跨度的统计分析
- 适用于项目周期、特定事件的时间统计

#### 7.2.3 统计维度

**按标签统计**：

- 显示各标签的总时长和占比
- 支持多级标签的层级统计
- 标签排序（按时长、按名称）

**按活动类型统计**：

- 显示不同活动类型的时长分布
- 支持活动类型的层级分类统计

**按时段统计**：

- 统计不同时间段的活动分布
- 例如：上午（6-12点）、下午（12-18点）、晚上（18-24点）
- 帮助发现个人效率高峰时段

**按工作日/周末统计**：

- 对比工作日和周末的时间使用差异
- 分析工作与生活平衡情况

#### 7.2.4 图表类型

**饼图（Pie Chart）**：

- 用途：展示各标签/活动类型的时长占比
- 特性：
  - 不同标签使用可区分的配色
  - 显示百分比和实际时长
  - 点击扇区可查看详细数据
  - 支持标签筛选（隐藏/显示特定标签）
- 适用场景：日/周/月总体时间分配一览

**柱状图（Bar Chart）**：

- **垂直柱状图**：
  - 用途：展示时间趋势变化
  - X轴：时间维度（日期、周数、月份）
  - Y轴：时长（小时）
  - 支持多标签对比（堆叠或并列柱状图）
  - 适用场景：周统计中每日时长对比、月统计中逐日趋势、年统计中逐月趋势
- **水平柱状图**：
  - 用途：展示标签排名
  - X轴：时长
  - Y轴：标签名称
  - 按时长从高到低排序
  - 适用场景：快速查看时间投入最多的标签

**折线图（Line Chart）**：

- 用途：展示时间使用趋势
- 多条折线可对比多个标签的变化趋势
- 支持平滑曲线显示
- 适用场景：月度/年度趋势分析

**堆叠面积图（Stacked Area Chart）**：

- 用途：展示各标签时长的累积分布
- 直观显示总时长变化和各标签占比变化
- 适用场景：周/月/年的整体时间结构分析

#### 7.2.5 统计指标

**基础指标**：

- 总记录时长
- 平均每日时长
- 记录条数
- 最长/最短单条记录

**对比指标**：

- 环比增长率（与上一周期对比）
- 同比增长率（与去年同期对比）
- 目标完成率（实际时长 vs 目标时长）

**分布指标**：

- 标签时长占比
- 时段分布占比
- 工作日/周末占比

**效率指标**：

- 深度工作时长（可标记特定标签为深度工作）
- 碎片化指标（平均单条记录时长）
- 专注度评分（基于记录连续性）

#### 7.2.6 交互功能

**图表交互**：

- 点击图表元素查看详细数据
- 长按显示精确数值
- 缩放和平移查看数据
- 双指缩放调整时间范围

**数据筛选**：

- 按标签筛选（多选）
- 按活动类型筛选
- 按时间范围筛选
- 快速切换时间维度（日/周/月/年）

**数据导出**：

- 导出图表为图片
- 导出统计数据为CSV/Excel
- 生成统计报告（PDF）
- 分享统计结果到社交平台

**快速跳转**：

- 从统计图表跳转到对应的时间块视图
- 从图表元素跳转到具体记录列表
- 快速切换不同时间周期的统计

#### 7.2.7 统计页面布局

**顶部导航区**：

- 时间维度选择器（日/周/月/年/自定义）
- 日期选择器
- 快速跳转按钮（今天、本周、本月）

**概览卡片区**：

- 总时长大数字显示
- 环比变化百分比和趋势箭头
- 记录条数
- 目标完成率进度条

**主图表区**：

- 饼图（时间分配占比）
- 柱状图（趋势变化）
- 支持左右滑动切换图表类型

**详细数据区**：

- 标签排名列表
- 每个标签显示：名称、时长、占比、环比变化
- 点击标签进入该标签的详细统计

**洞察建议区**（可选）：

- 基于数据的智能分析
- 例如："本周学习时长比上周增加15%，继续保持！"
- "上午9-11点是你的高效时段，建议安排重要任务"

#### 7.2.8 特色功能

**对比分析**：

- 选择两个时间段进行对比
- 并排显示两个时间段的统计图表
- 高亮差异变化
- 适用场景：本月 vs 上月、本周 vs 上周

**标签深度分析**：

- 单击某个标签进入详细统计页
- 显示该标签的：
  - 历史趋势（折线图）
  - 时段分布（热力图）
  - 记录列表（所有该标签的记录）
  - 关联标签（经常同时出现的其他标签）

**统计报告生成**：

- 自动生成周报、月报
- 包含关键数据、图表、洞察建议
- 支持自定义报告模板
- 可设置自动定期生成

### 7.3 目标管理（Goals）

#### 7.3.1 功能定位

通过设定时长目标，帮助用户有计划地分配时间，实现时间使用的优化。

#### 7.3.2 目标类型

**按标签设定目标**：

- 为特定标签设定时长目标
- 例如："每周学习20小时"、"每天阅读1小时"

**按时间周期**：

- 每日目标
- 每周目标
- 每月目标
- 自定义周期（如每季度）

**目标维度**：

- 最少时长：确保某活动的最低投入
- 最多时长：限制某活动的时间上限
- 精确时长：追求准确的时间分配

#### 7.3.3 目标追踪

**进度显示**：

- 实时显示目标完成进度（百分比、进度条）
- 剩余时长提示
- 预计完成时间（基于当前速度）

**历史记录**：

- 目标完成率统计
- 历史目标达成趋势
- 连续达成天数记录

#### 7.3.4 目标管理

- 创建新目标
- 编辑目标参数
- 暂停/恢复目标
- 删除目标
- 目标模板（快速创建相似目标）
